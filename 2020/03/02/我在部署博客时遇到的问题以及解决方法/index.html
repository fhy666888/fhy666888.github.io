我在部署博客时遇到的问题以及解决方法

发表于 2020-03-02 | 更新于: 2020-04-13 | | 阅读数

​ 功夫不负有心人，花了整整两到三天把hexo博客部署好，这一过程可以说是崎岖坎坷，遇到 了许多磕磕绊绊。接下来我就说说我在部署博客时遇到的问题以及解决方法吧，可供参考！

​ 具体的博客的本地搭建，动手写博文，Hexo博客乔迁到Github的步骤方法我在这就不细说 了，你们可以参照这个博客：http://www.yangkai123.top/

  • 博客的本地搭建

    Hexo的环境配置并没什么问题，但是需要注意的一点是，每次重启电脑或重新打开CMD窗口时，一定要执行：

    ​ cd blog

    ​ hexo init

    ​ 否则会有错误，

    ​ 之后，直接运行hexo s可以预览到效果，

    ​ 所以hexo博客的本地搭建很快就成功了。

部署博客中的遇到的各种问题

  • 添加图片

​ 首先是添加图片，我上网查了许多资料，具体有两种方法：

​ 第一种方法如下：

  1. 安装插件 npm install hexo-asset-image –save
  2. 把主页配置文件_config.yml 里的post_asset_folder:这个选项设置为true

​ 3. 安装完成后，再运行hexo new “filename”来生成md博文时，/source/_posts文件夹内除了 filename.md文件还有一个同名的文件夹

  1. 在title.md中引入图片时，先把图片复制到title这个文件夹中，然后只需要在title.md中按照 markdown的格式引入图片

    第二种方法如下：

  2. 在Typora文件->偏好设置，图片插入的地方选择“拷贝到自定义路径”，因为其他几种方式都是拷贝到当前位置的assets或其他子目录下的，这样hexo肯定找不到。

  3. 在source下建一个assets文件夹，并把拷贝路径设置到这里。

这样导致hexo能找到你网站的根目录下的assets，然而你图片在source目录下，这就尴尬了，我也 卡在这里蛮久的。

  1. 下面点击格式->图像>设置图片根目录，把目录设置到source，问题就解决啦，你会看到这样的路径

​ (注：我觉得第二种方法比较巧妙吧，设置起来也比较方便！推荐使用！)

  • 添加评论

​ 添加了图片，也要有评论，可用的评论系统大概有：

​ 来必力：https://livere.com （来自韩国，使用邮箱注册。）

​ 畅言： http://changyan.kuaizhan.com （安装需要备案号。不太好用。）

​ Gitment： https://github.com/imsun/gitment （有点小bug，比如说每次需要手动初始化， 登录时会跳到主页。。）

​ Valine: https://github.com/xCss/Valine (简洁，方便，好用的评论系统）

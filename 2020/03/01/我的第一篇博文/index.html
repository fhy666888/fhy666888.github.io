如何使用Next主题？

我们通常初始化hexo博客，并把它部署到网站上后，它的整个页面就完成了。但是它并不美观有 趣，别人在访问你的github的同时也会感到枯燥与无趣。hexo里有许多主题，它可以给我们的博 客”化妆”，使得我们的页面更加生动。接下来我就介绍Next主题的几种简单用法。

  • 安装next

下载主题时有两种方法：

  1. 在终端窗口下，定位到 Hexo 站点目录下。使用 Git checkout 代码：

2.下载稳定版本：前往NexT版本发布页面

（注：就我本人经历而言，第一个方法我试了好多遍，都没安装的上，不知道哪里出了问题，最 终我采用了第二种方法)

  • 启用主题

    打开站点配置文件(_config.yml)， 找到them字段，并将其值更改为next。

    theme: next

​ 到此，NexT 主题安装完成。下一步我们将验证主题是否正确启用。在切换主题之后、验证之 前，我们最好使用 hexo clean 来清除 Hexo 的缓存。

  • 验证主题

    首先启动 Hexo 本地站点，并开启调试模式（即加上 --debug），整个命令是 hexo s --debug。 在服务启动的过程，注意观察命令行输出是否有任何异常信息，如果你碰到问题，这些信息将帮助他人更好的定位错误。

    当命令行输出中提示出：INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.

    此时即可使用浏览器访问http://0.0.0:4000/ ，检查站点是否正确运行。

  • 主题设定

    *选择 Scheme*:

​ Scheme 的切换通过更改主题配置文件，搜索 scheme 关键字。 你会看到有三行 scheme 的配 置，将你需用启用的 scheme 前面注释#去除即可。

​ 例：#scheme: Muse

​ #scheme: Mist

​ scheme: Pisces
